Mapa Conceptual: Arquitectura SOA

Evolución, Conceptos Clave y Tecnologías de Integración

Antecedente
Arquitecturas Monolíticas

Representan el modelo tradicional de desarrollo de software donde todos los componentes funcionales de una aplicación (interfaz, lógica y datos) están integrados en una única unidad lógica y de despliegue.

  • Acoplamiento Fuerte: Los cambios en una parte afectan a todo el sistema.
  • Escalabilidad Vertical: Se requiere aumentar recursos de todo el servidor, no de partes específicas.
  • Riesgo: Un fallo en un módulo puede tumbar toda la aplicación.
↓
Paradigma Central
SOA (Service-Oriented Architecture)

Estilo de diseño de software donde las aplicaciones se componen de servicios discretos que se comunican entre sí a través de una red.

Servicios: Unidades de lógica de negocio autónomas, independientes y reutilizables.
ESB (Enterprise Service Bus): El "sistema circulatorio" de SOA. Un middleware que gestiona la comunicación, transformación de protocolos y enrutamiento entre servicios.
↓
Interfaz
APIs (Application Programming Interfaces)

El contrato formal que permite que los servicios consuman capacidades de otros servicios de forma estandarizada.

Tecnología Descripción Formato de Datos
SOAP Protocolo estricto basado en estándares XML. Muy común en SOA clásico. XML
API REST Estilo arquitectónico ligero que utiliza los estándares de la web (HTTP). JSON / XML / Texto
Operaciones
Métodos HTTP (Verbos)

En el contexto de API REST, definen la intención de la petición hacia un recurso:

  • GET: Recuperar información de un recurso.
  • POST: Crear un nuevo recurso.
  • PUT / PATCH: Actualizar un recurso existente de forma total o parcial.
  • DELETE: Eliminar un recurso.
